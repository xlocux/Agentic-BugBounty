Design — Flusso Wizard

Come la UI guida l'utente dall'inizio alla submission, con controllo su ogni fase

1
Target Setup
Incolla URL programma → nome auto-filled → Create → sync scope + intel automatico
⊕ OBBLIGATORIO
2
Asset Selection
La UI mostra gli asset in-scope con checkbox. Tu selezioni quali testare (es. *.duckduckgo.com ✓, spread.duckduckgo.com ✗). Selezione salvata in session.json.
⊕ OBBLIGATORIO — il pipeline non parte senza almeno 1 asset selezionato
3
⏸ Plan Approval — Explorer
Pipeline scrive piano in session.json → UI mostra: "Eseguirò surface mapping su 3 assets. Operazioni: [✓ deps scan] [✓ endpoint map] [○ secret scan — opzionale]". Tu approvi/modifichi → pipeline riprende.
⊕ deps scan: obbligatorio ○ secret scan: opzionale ○ git intel: opzionale
4
⏸ Plan Approval — Researcher
Pipeline analizza gli asset selezionati e scrive: "Per *.duckduckgo.com eseguirò: [✓ AUTH] [✓ INJECT] [✓ ACCESS] [○ MEDIA — no file upload in scope] [○ INFRA — non in scope]". Tu scegli i domini da ricercare.
⊕ AUTH, INJECT: obbligatori ○ CLIENT, ACCESS, MEDIA, INFRA: opzionali
5
Execution Monitor
Live progress: quale domain sta girando, findings confermati man mano, possibilità di pausare/skippare un domain a runtime.
6
⏸ Findings Review + Triage Approval
Prima del triage: UI mostra i finding confermati con severity. Tu puoi: rimuovere falsi positivi, aggiungere note, cambiare severity. Poi approvi il triage.
7
Submit
Preview report H1-ready, un click per inviare. Feedback di conferma.
Legenda: ⊕ obbligatorio ○ opzionale/selezionabile ⏸ checkpoint — pipeline pausa, aspetta approvazione UI
