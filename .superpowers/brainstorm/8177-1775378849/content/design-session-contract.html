Design — Session Contract

Il file targets/{name}/session.json è il canale bidirezionale tra pipeline e UI

Pipeline scrive →
// Checkpoint: asset selection
{
 "status": "awaiting_assets",
 "phase": "setup",
 "asset_type": "browserext",
 "available_assets": [
  {"id":"ext-main","label":"Extension main","mandatory":true},
  {"id":"content-scripts","label":"Content scripts","mandatory":false}
 ]
}

// Checkpoint: plan approval
{
 "status": "awaiting_approval",
 "phase": "researcher",
 "asset_type": "browserext",
 "plan": [
  {"id":"postmessage","label":"postMessage vulns","mandatory":true,"reason":null},
  {"id":"dom_xss","label":"DOM XSS","mandatory":true,"reason":null},
  {"id":"content_script","label":"Content script injection","mandatory":false,"reason":"requires user interaction"},
  {"id":"supply_chain","label":"Supply chain","mandatory":false,"reason":"no npm deps found"}
 ],
 "context": "Found 3 content scripts, 2 background workers, 1 popup"
}

// Progresso (UI lo legge per il monitor)
{
 "status": "running",
 "phase": "researcher",
 "current_op": "postmessage",
 "completed_ops": ["dom_xss"],
 "findings_so_far": 2
}
← UI scrive (risposta)
// Risposta asset selection
{
 "status": "assets_selected",
 "selected_assets": ["ext-main"]
}

// Risposta plan approval
{
 "status": "approved",
 "approved_ops": [
  "postmessage",
  "dom_xss",
  "content_script"
 ]
 // supply_chain non incluso → skippato
}

// Richiesta skip a runtime
{
 "status": "skip_requested",
 "skip_op": "content_script"
}
Domini per asset_type
webapp: AUTH INJECT CLIENT ACCESS MEDIA INFRA
browserext: postMessage DOM-XSS content-script supply-chain permissions
mobileapp: deep-links intent storage network crypto
executable: memory-corruption binary-analysis input-validation priv-esc
Meccanismo: Pipeline scrive session.json con status: "awaiting_*" poi entra in polling loop (ogni 500ms). UI riceve notifica via SSE (server fa watch su session.json), mostra il piano, utente approva → UI scrive status: "approved" → pipeline riprende. Schema formale in schemas/session.schema.json.
