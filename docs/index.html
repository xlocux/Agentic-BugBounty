Bug Bounty Vulnerability Map

Core vulnerability families across all asset types, with the operational layer that turns the framework from a prompt collection into a validated pipeline.

Framework overview
The map organizes vulnerabilities by family inside a stronger workflow: machine-readable target setup, bundle validation, triage validation, local rendering, logs, and regression tests.
Injection Server-side Auth Access control Client-side Protocol / infra
Injection and parser abuse
Classic input-to-interpreter bugs remain first-class targets in whitebox and blackbox workflows.
SQLi XSS and DOM XSS NoSQL and command injection SSTI / CSTI XXE / SSRF Deserialization / prototype pollution
Server-side logic and architecture
Business logic, upload abuse, file parsing, and cloud-facing mistakes often need deeper proof than the first candidate suggests.
File upload bypass Traversal / LFI / RFI Race conditions / response abuse Business logic flaws Mass assignment Cloud misconfig / exposed debug
Authentication and session
Severity is often overclaimed here, so the triage layer explicitly recalculates evidence-backed impact and CVSS.
Broken auth / JWT 2FA / reset bypass Session fixation / hijack OAuth / SSO flaws CSRF / login CSRF Enumeration / brute force gaps
Access control
IDOR, forced browsing, and privilege escalation stay central because they are reproducible and benefit from precise target scope metadata.
IDOR Horizontal privesc Vertical privesc Forced browsing Verb tampering Open redirect chains
Client-side and browser surfaces
Web and extension-facing bugs grouped together because the framework spans browsers, message passing, and extension privilege boundaries.
Clickjacking postMessage Dangling markup / JSONP Manifest permissions uXSS / web_accessible Remote config / data exfil
Protocol, infra, and supply chain
These modules often need stronger correlation between target context, runtime evidence, and external environment assumptions.
HTTP smuggling Cache poisoning CORS / GraphQL / WebSocket Subdomain takeover Dependency confusion CI/CD injection / repo secrets
Mobile and binary focus areas
APK/IPA and executable workflows share the same contract and output system as all other asset types.
Hardcoded secrets Insecure storage Deep links / exported components Buffer / heap overflow Format string / UAF TOCTOU / integer overflow
Bypass and evasion support
Bypass packs are loaded on-demand when payloads are blocked. The workflow shows exactly where they attach and how output is validated.
Encoding chains XSS / SQLi evasion SSRF / auth / WAF bypass Null byte tricks Unicode confusion Magic bytes / double encoding
Operational layer
These are the pieces that turn the repository from a prompt collection into a reliable, testable framework.
